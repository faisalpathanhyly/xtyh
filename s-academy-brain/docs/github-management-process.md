# GitHub Management Process

## Recommended branches

```text
main        = approved stable repository
staging     = work ready for review/testing
feature/*   = new additions or migrations
```

## Simple workflow

```text
Create/update files
↓
Review changes
↓
Commit with clear message
↓
Open pull request
↓
Approve
↓
Merge into main
```

## Commit message examples

```text
Add repository foundation files
Add Introduction to Multifamily metadata
Migrate Halo Overview chapter placeholders
Update question bank schema
Add content governance documentation
```

## Pull request checklist

- [ ] File names follow repo standards
- [ ] Metadata is complete
- [ ] No secrets included
- [ ] Markdown is readable
- [ ] JSON is valid
- [ ] Source is documented
- [ ] Owner and status are clear
