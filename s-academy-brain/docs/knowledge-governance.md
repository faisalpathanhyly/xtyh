# Knowledge Governance

## Purpose

Governance keeps the repository trustworthy, reusable, and maintainable.

## Required ownership

Each knowledge asset should have:

- Owner
- Status
- Version
- Source
- Last updated date
- Review status

## Lifecycle

```text
draft
↓
in-review
↓
approved
↓
published
↓
archived
```

## Review standards

A reviewer should check:

- Accuracy
- Completeness
- Relevance
- Reusability
- AI-readability
- Source traceability

## When to archive

Archive an asset when:

- It is outdated
- It has been replaced
- It is no longer used
- It contains retired product/process information
